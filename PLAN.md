## 2단계: 데이터 수집 + 커스텀 YOLO 학습

### 왜 필요한가
현재 웨이스트/높이/헛치기 폴트는 **셔틀콕 실제 위치**가 있어야 정확.
지금은 손목 위치로 근사 중 → 오탐 발생 가능.

### 커스텀 YOLO 클래스
| 클래스 | 용도 |
|--------|------|
| `shuttlecock` (0) | 웨이스트 / 높이 / 헛치기 폴트 |
| `racket_head` (1) | 샤프트 폴트 정밀화 |

### 수집 목표 데이터
| 폴트 유형 | 목표 수량 |
|----------|----------|
| 정상 서브 | 100개 |
| 웨이스트 폴트 | 50개 |
| 높이 폴트 | 30개 |
| 샤프트 폴트 | 30개 |
| 쉐이크 폴트 | 30개 |
| 발 폴트 | 30개 |
| 헛치기 | 20개 |

### 라벨링 워크플로우
```
1. python extract_frames.py          # 5프레임마다 1장 추출
2. Roboflow 업로드 → 바운딩박스 라벨링
   - shuttlecock: 셔틀콕 전체
   - racket_head: 라켓 헤드 부분
3. YOLOv8 포맷으로 export
4. python train_model.py dataset/data.yaml --epochs 150
5. python train_model.py dataset/data.yaml --val --model runs/train/badminton_v1/weights/best.pt
```

### 학습 목표 성능
- mAP50 ≥ 0.85 (shuttlecock)
- mAP50 ≥ 0.80 (racket_head)

### 촬영 세팅 (BWF 심판 기준)
```
카메라: 선수 정측면 90도, 허리 높이
거리:   전신(머리~발) + 좌우 여유
fps:    60fps 이상
해상도: 1080p 이상
```

## 3단계: 라이브 검출기

### 현재 배치 방식과 차이점
| 항목 | 배치 (현재) | 라이브 |
|------|------------|--------|
| 입력 | .mov 파일 | 웹캠 실시간 |
| 임팩트 감지 | 전체 영상 통계 | 슬라이딩 윈도우 |
| 허리선 보정 | 첫 90프레임 | 처음 2초 준비 자세 |
| 결과 | 영상 저장 | cv2.imshow() 즉시 표시 |
| 패스 수 | 3패스 | 1패스 |

### 라이브 상태 머신
```
IDLE (대기)
  → (손목 속도 증가 감지)
SERVING (서빙 중) — 허리선/높이선/쉐이크 실시간 표시
  → (속도 피크 = 임팩트)
RESULT (판정) — FAULT / OK 2초간 표시
  → (자동 리셋)
IDLE
```
