## 각 폴트 상세 판별 로직

### 웨이스트 폴트 (9.1.6)
```
판별 시점: 임팩트 프레임
조건: shuttle_y > waist_y (셔틀콕이 허리선 아래)
허리선: 어깨~엉덩이 사이 60% (갈비뼈 최하단 추정)
```

### 높이 폴트 (9.1.6.2)
```
판별 시점: 임팩트 프레임
조건: shuttle_y > height_threshold_y (1.15m 기준선 아래)
1.15m선: 신체 비율로 픽셀 변환 (1회 보정 후 고정)
```

### 샤프트/오버헤드 폴트 (9.1.7)
```
판별 시점: 임팩트 프레임
조건: racket_head_y < wrist_y (헤드가 손목보다 위 = 위를 향함)
```

### 쉐이크 폴트 (9.1.8)
```
서브 시작 정의: 셔틀콕과 라켓헤드가 가까워지는 순간 (proximity 기반)
판별 시점: 셔틀콕 ↔ 라켓헤드 거리가 임계값 이하인 프레임
조건: 해당 순간 라켓헤드의 x방향 속도 > 임계값
     = 공과 라켓이 가까울 때 라켓이 멈추지 않고 흔들리면 폴트
촬영: 오른손잡이 → 오른쪽 90도 촬영 (flip 증강으로 왼쪽도 자동 학습)
```

### 빽지체 폴트 (9.1.8)
```
판별 시점: 서브 시작(serve_start) ~ 임팩트 전 구간
조건: 라켓헤드의 x 이동 방향이 뒤로 전환되는 순간 감지
     = 앞으로 가다가 뒤로 빠지면 폴트
주의: 백스윙(서브 전 뒤로 빼는 동작) 끝난 후부터 체크
```

### 선밟기 폴트 (9.1.4)
```
판별 시점: 서브 시작(serve_start) 프레임
조건: ankle_y >= service_line_y (발목이 서비스 라인에 닿거나 넘음)
서비스 라인: 코트 라인 자동 감지 or 수동 설정
```

### 발 고정 폴트 (9.1.4)
```
판별 시점: 서브 시작 ~ 임팩트 전 구간
조건: 발목 이동량 > 임계값 (80px, 5프레임 연속)
     OR 발이 코트 표면에서 떨어짐 (도약 감지)
```

### v4 라벨링 계획 (진행 중)
**셔틀콕 우선 영상:**
- 허리선넘기 50장 ← 임팩트 순간 셔틀콕
- 정상 50장 ← 다양한 서브 셔틀콕
- 앞45도 50장 ← 라켓 앞면에서 셔틀콕
- 쉐이크폴트 50장 ← 셔틀콕+라켓 근접

**라켓헤드 우선 영상:**
- 앞45도 50장 ← 납작함 없이 라켓면 보임
- 뒤45도 50장 ← 백스윙 라켓헤드
- 오버헤드 50장 ← 다양한 라켓 방향
- 쉐이크폴트 50장 ← 흔들리는 라켓헤드

### 서브 경계 감지 로직 개선 계획
```
현재: 손목 속도 기반 임팩트 감지
목표: 셔틀콕 ↔ 라켓헤드 근접도 기반

서브시작 = 셔틀콕+라켓헤드 거리가 임계값 이하로 처음 진입
임팩트   = 두 객체 거리 최솟값
서브끝   = 임팩트 후 셔틀콕이 빠르게 멀어지는 순간

※ 감지 실패 시 손목 속도 기반으로 자동 폴백
```

### 라벨링 워크플로우
```bash
# 1. 프레임 추출 (5프레임마다 1장)

python train_model.py dataset/data.yaml --epochs 150

# 5. 검증
python train_model.py dataset/data.yaml --val \
    --model runs/train/badminton_v1/weights/best.pt
```

## 촬영 가이드

### 카메라 세팅
```
위치: 선수 오른쪽 90도, 허리 높이
거리: 전신(머리~발) + 좌우 여유
fps:  60fps 이상
해상도: 1080p 이상
```

### 좌우 방향 문제
- 오른손잡이 → 오른쪽 90도 촬영
- 왼손잡이 → 왼쪽 90도 촬영
- YOLOv8 수평 반전(fliplr=0.5) 증강으로 반대 방향도 자동 학습됨
- → 한쪽만 찍어도 양쪽 방향 모두 인식 가능

### 쉐이크 폴트 촬영 시
- 서브 준비 → 라켓을 좌우로 크게 흔드는 동작 포함
- 셔틀콕과 라켓헤드가 가까워지는 순간이 선명하게 보여야 함
- 20~30개 폴트 동작 권장
